# docs/refactor 使用说明

本目录包含重构评审文档、执行计划与进度记录（手工维护）。

## 1. 目录约定

- `refactor-design.md`：重构设计方案（架构、分层、接口、验收口径）
- `steps-and-estimates.md`：模块重构步骤清单（按模块优先级 + 预计工时）
- `progress.md`：进度追踪表（状态/完成度/负责人/实际耗时）
- `alerts.md`：风险预警面板（按规则手工判定并记录）
- `gantt.md`：里程碑甘特图（关键节点）
- `logs/project-progress-log.md`：项目级进度日志（由我们持续追加）
- `templates/`：MR 模板、评审纪要模板、模块重构日志模板

## 2. 进度更新方式

1. 更新进度：直接编辑 `progress.md`（模块状态/完成百分比/负责人/实际耗时）。
2. 记录日志：在 `logs/project-progress-log.md` 追加一条“本次进度变化与关键信息”。
3. 更新预警：按 `alerts.md` 的触发规则，手工标记是否触发预警与原因。

## 3. 提交约束

本仓库不做任何“自动提交/自动暂存/自动生成并写入”的强制行为；所有进度与日志均以人工审核为准。
