# 安全代码审查清单（OWASP Top 10 对齐）

## A01 访问控制失效

- [ ] 所有写接口均要求登录态，未登录返回统一业务码
- [ ] 资源访问采用“当前用户上下文”校验，不信任前端传入的用户 ID
- [ ] 网关/服务侧不存在“误放行”白名单路径

## A02 加密机制失效

- [ ] 配置文件未出现明文密钥、令牌、口令
- [ ] 敏感配置均来自受控配置中心/密钥管理
- [ ] 敏感数据落库/传输满足最小可见与脱敏要求

## A03 注入

- [ ] SQL/排序字段使用白名单，不使用字符串拼接
- [ ] 模板渲染与前端 DOM 更新避免注入未转义输入
- [ ] JSON 反序列化与表达式执行路径受限

## A04 不安全设计

- [ ] 鉴权、幂等、异常处理有统一策略并在公共层复用
- [ ] 关键业务链路有失败回滚/补偿策略

## A05 安全配置错误

- [ ] CORS 仅允许受信任域名，不使用 `*` + credentials
- [ ] 默认暴露端点最小化（Actuator、调试接口）
- [ ] 生产环境禁用调试开关与测试账号

## A06 易受攻击和过时的组件

- [ ] 执行依赖安全扫描（Dependency-Check/SCA）
- [ ] 高危 CVE 有升级或临时缓解策略

## A07 身份认证与会话管理失效

- [ ] Session/Cookie 不出现在 URL 与日志
- [ ] 鉴权失败/越权返回统一错误码，且日志不泄露凭据

## A08 软件和数据完整性故障

- [ ] 外部依赖来源可信，构建产物可追溯
- [ ] 关键配置变更可审计、可回滚

## A09 安全日志与监控失败

- [ ] 日志包含 traceId，可关联请求与异常
- [ ] 日志不记录 Cookie/Authorization/token/密码明文
- [ ] 告警规则覆盖登录失败、权限异常、异常峰值

## A10 服务端请求伪造（SSRF）

- [ ] 外部 URL 调用有域名白名单与超时限制
- [ ] 不允许用户可控 URL 直连内部网段
