# AI 客服需求分析文档（技术可行性优化版）

## 1. 业务背景与目标

### 1.1 项目愿景
基于 **GuliMail 微服务架构**，利用 **LangChain4j + 大语言模型 (LLM)** 技术，打造一个深度集成电商业务流程的“智能购物助手”。该助手旨在通过自然语言交互，为用户提供商品咨询、库存查询、秒杀导购及订单售后状态追踪等一站式服务，提升购物体验并降低基础咨询的人工成本。

### 1.2 核心 OKR (Objectives and Key Results)
*   **O1：提升购物转化率与用户体验**
    *   KR1：AI 助手平均响应延迟（首字输出）控制在 1 秒以内。
    *   KR2：导购意图（查询商品、推荐）的准确率达到 85% 以上。
*   **O2：实现业务流程自动化**
    *   KR1：售后咨询（订单状态、物流）的自动解答率达到 90%。
    *   KR2：实现与 Product、Order、Member、Seckill 等核心微服务的无缝数据打通。

## 2. 系统架构与技术栈

### 2.1 现有技术架构
*   **微服务框架**：Spring Cloud Alibaba (Nacos, Gateway, Feign, Sentinel)。
*   **AI 引擎**：LangChain4j 0.25.0+ (集成 DeepSeek/OpenAI 接口)。
*   **会话管理**：Spring Session + Redis (支持多模块共享登录状态)。
*   **通信协议**：Server-Sent Events (SSE) 实现打字机流式响应。
*   **数据来源**：通过 Feign 客户端动态调用各业务微服务接口。

### 2.2 核心业务链路
1.  **用户侧**：前端 Thymeleaf/Vue 通过 SSE 与 `gulimail-ai` 通信。
2.  **AI 层**：`AiChatController` 接收请求 -> `MallAssistant` (AI Service) -> LLM (DeepSeek)。
3.  **工具层**：LLM 根据需求触发 Tool Call -> `ProductAiTools` -> Feign 远程调用 -> 业务数据返回。

## 3. 功能需求（优先级排序）

### 3.1 核心导购功能 (P0)
*   **商品查询**：支持按关键词、分类、品牌搜索商品。
*   **详情展示**：根据 `skuId` 返回商品规格、价格、详情图链接（Markdown 格式渲染）。
*   **动态库存/秒杀校验**：推荐商品时，必须实时调用 `ware` 和 `seckill` 服务确认是否有货及优惠，防止无效推荐。

### 3.2 智能售后查询 (P0)
*   **订单追踪**：识别用户身份后，自动列出最近订单及状态（待付款、已发货等）。
*   **物流状态**：对接物流微服务（如有），告知包裹大致位置。

### 3.3 交互体验优化 (P1)
*   **流式打字机**：后端切片发送，前端字符队列渲染，消除 LLM 思考过程中的卡顿。
*   **多轮对话记忆**：支持 ≥5 轮的历史对话上下文，能够理解“那个商品多少钱？”中的指代关系。
*   **Markdown 富媒体**：支持在对话中显示商品图片、链接按钮、表格对比。

### 3.4 异常处理与兜底 (P1)
*   **服务降级**：当 LLM API 故障或超时时，返回友好的固定话术（兜底方案）。
*   **意图拦截**：敏感词过滤，禁止回答政治、暴力或非业务相关问题。

### 3.5 延期/移除功能 (Out of Scope)
*   **语音交互**：由于缺乏语音网关和低延迟 ASR/TTS 环境，本阶段不予实现。
*   **人工接管**：项目目前无客服管理后台，暂不实现即时转人工。
*   **多语言扩展**：受限于 Prompt 长度和翻译准确度，首期仅支持中文。

## 4. 非功能需求

### 4.1 性能指标
*   **并发能力**：单实例支持并发 50-100 QPS（受外部 LLM API 频率限制）。
*   **响应耗时**：P99 首字输出延迟 ≤1.5s；整体生成延迟（带 Tool Call） ≤5s。

### 4.2 安全与稳定性
*   **鉴权保护**：AI 接口必须经过 Gateway 的 JWT 校验，禁止未登录用户恶意刷接口。
*   **密钥安全**：所有 API Key 必须存放在 Nacos 配置中心，禁止硬编码。
*   **会话隔离**：确保不同用户的对话上下文在 Redis 中完全隔离。

## 5. 接口集成方案

### 5.1 核心 Feign 接口清单
*   **Product 服务**：`/api/item/{skuId}` (商品详情)、`/api/category/list` (分类列表)。
*   **Order 服务**：`/order/order/listWithItem` (订单列表)。
*   **Ware 服务**：`/ware/waresku/hasstock` (库存校验)。
*   **Seckill 服务**：`/seckill/sku/info/{skuId}` (秒杀优惠)。

### 5.2 通信协议规范
*   **Request**: `GET /api/ai/chat/stream?msg=xxx` (携带 GULISESSION Cookie)。
*   **Response**: `text/event-stream`，数据块按 `chunk` 发送，以 `[DONE]` 结束。

## 6. 实施路线图

### 6.1 第一阶段：MVP 最小可行性产品（当前已基本完成）
*   实现流式对话、基础商品查询和订单查询 Tool 调用。
*   完成 Nacos 配置和 Redis Session 集成。

### 6.2 第二阶段：体验与准确性调优（下周目标）
*   **Prompt 工程优化**：精细化 `@SystemMessage`，解决 AI 工具调用循环和幻觉问题。
*   **前端打字机优化**：实现字符缓冲队列，让视觉效果更丝滑。
*   **Tool 返回数据精简**：优化 Feign 返回的 JSON 结构，减少 Token 消耗。

### 6.3 第三阶段：业务闭环（未来展望）
*   **用户画像集成**：根据 Member 服务的用户标签（如“高消费”、“新用户”）调整推荐话术。
*   **简单工单创建**：在 Member 模块增加“意见反馈”表，支持 AI 自动生成反馈单。

## 7. 验收标准

1.  **功能验证**：用户询问“我的订单在哪”，AI 能准确说出最近一笔订单的状态和商品。
2.  **性能验证**：从发送消息到看到第一个字显示，平均耗时低于 1.5 秒。
3.  **安全验证**：未登录用户直接访问 `/api/ai/chat/stream` 应当被拦截或返回 401。
4.  **体验验证**：对话内容包含商品图片和可点击的详情页链接。
